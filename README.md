# Benchmark Supplement

This repository is the benchmark workspace used for the dissertation's
comparison of LLM-based and classical feature-engineering pipelines on tabular
datasets.

It is intended as a **code-and-results supplement**, not as a full archival
snapshot of the original local workspace. Large generated artifacts, local
virtual environments, historical feature dumps, and other machine-specific
files are excluded from version control.

## What This Repository Contains

- benchmark runner wrappers for each compared method
- shared dataset handling and evaluation code
- cleaned input datasets used in the benchmark
- final result summaries and selected analysis outputs
- patch snapshots and patch notes for benchmark-specific changes

This repository does **not** aim to preserve every raw intermediate artifact
produced during experimentation.

## Repository Layout

- `datasets/`
  - source benchmark datasets
- `results/`
  - evaluation outputs, benchmark reports, and final analysis tables
- `env_specs/`
  - frozen package snapshots for each local benchmark environment
- `tools/`
  - analysis and utility scripts used during the dissertation project
- `patches/`
  - patch snapshots for benchmark-specific changes applied to local upstream
    clones
- `repos/`
  - local upstream method clones used during benchmarking
- `dataset_utils.py`
  - shared dataset metadata, cleaning, validation, and output-path logic
- `evaluator.py`
  - benchmark evaluator and report generator
- `run_ownm.py`
  - in-repo iterative LLM feature-generation method
- `run_caafe.py`
  - CAAFE wrapper
- `run_llmfe.py`
  - LLM-FE wrapper
- `run_llmfe_eff.py`
  - efficiency-audit variant of the LLM-FE wrapper
- `run_featllm.py`
  - FeatLLM wrapper
- `run_openfe.py`
  - OpenFE wrapper
- `THESIS_PATCH_NOTES.md`
  - concise record of material benchmark-integrated changes

## Benchmark Scope

The compared methods are:

- `ownm`
  - in-repo LLM method
  - supports classification and regression
  - token logging enabled
- `caafe`
  - classification only
  - token logging enabled
- `llmfe`
  - supports classification and regression through the wrapper path
  - token logging enabled
- `featllm`
  - classification only
  - token logging enabled
- `openfe`
  - non-LLM baseline
  - no token logging

The benchmark should be interpreted as a **benchmark-integrated comparison**:
several wrappers and local upstream clones were patched to make the methods run
reliably under a shared contract. Where those patches materially affect
behavior, they are documented in `THESIS_PATCH_NOTES.md` and the corresponding
patch files under `patches/`.

## Key Outputs

Important benchmark outputs live under `results/`, including:

- `results.csv`
  - row-level evaluator outputs
- `results_mean.csv`
  - method-by-dataset aggregated results
- benchmark report text files
- token-attribution tables
- DKU and feature-stability summaries used in the dissertation analysis

Feature matrices generated by method runs are normally written under:

```text
features/<method>/<dataset>/<dataset>-<run_index>/<dataset>_features.csv
```

with matching metadata sidecars:

```text
features/<method>/<dataset>/<dataset>-<run_index>/<dataset>_meta.json
```

These generated feature outputs are local benchmark artifacts and are not part
of the intended lightweight GitHub-facing supplement.

## Common Commands

Run one dataset with `ownm`:

```powershell
.\venvs\venv_ownm\Scripts\python.exe .\run_ownm.py --dataset diabetes
```

Run multiple iterations:

```powershell
.\venvs\venv_ownm\Scripts\python.exe .\run_ownm.py --dataset diabetes --loop-iterations 10
```

Run a full method benchmark:

```powershell
.\venvs\venv_openfe\Scripts\python.exe .\run_openfe.py --all-datasets --loop-iterations 10
```

Evaluate saved feature runs:

```powershell
.\venvs\venv_ownm\Scripts\python.exe .\evaluator.py --input_dir .\features --output_dir .\results
```

Attribute token usage from provider logs:

```powershell
.\venvs\venv_ownm\Scripts\python.exe .\tools\attribute_token_usage.py --features-dir .\features --output-dir .\results
```

## Notes on Patch Fidelity

This repository is not a strict paper-faithful reproduction of every compared
method. The emphasis was on running all methods inside a unified benchmark with
shared datasets, shared evaluation, and comparable outputs.
